# Listen — PSL Mobile (React Native)

React Native / Expo frontend for the Listen Pakistani Sign Language app.
Designed for on-device inference with zero backend dependency.

## Quick start (UI only — Expo Go)

```bash
cd frontend
npm install
npx expo start
```

Press `a` to launch in Android emulator (when Android Studio is set up),
`i` for iOS simulator, `w` for web, or scan the QR with **Expo Go** for a
quick UI preview on a real phone.

> **Note:** Real ML inference (TFLite + camera frame processors) requires
> a **dev build**, not Expo Go. See "Run on Android (real ML)" below.

## Run on Android (real ML)

The native ML pipeline uses **react-native-vision-camera** for frame
capture and **react-native-fast-tflite** for on-device inference. Both
require a native dev build.

### Prerequisites

1. **Android Studio** (or just the command-line SDK):
   - Install Android Studio: https://developer.android.com/studio
   - Open it, go to *More Actions → SDK Manager*, install:
     - Android SDK Platform 34
     - Android SDK Build-Tools 34
     - Android Emulator + at least one system image
2. **JDK 17**:
   ```bash
   brew install --cask zulu@17
   ```
3. Set up environment variables in `~/.zshrc`:
   ```bash
   export ANDROID_HOME="$HOME/Library/Android/sdk"
   export PATH="$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$PATH"
   ```
4. Verify:
   ```bash
   adb --version
   java -version    # should be 17.x
   ```

### Build & run

```bash
cd frontend
npx expo run:android       # plug in a device, OR start an emulator first
```

This produces a native APK with all native modules compiled in. From this
point, the camera screen runs the **real** TFLite words classifier
instead of the mock pipeline.

### Build & run (iOS, optional)

```bash
cd frontend
npx pod-install
npx expo run:ios
```

## ML pipeline

`src/ml/` ports the desktop Python pipeline from `src/inference/psl_words_v2.py`
with identical constants and math.

### Mock vs. real

- **Mock mode** (`pipeline.ts → mockTick`): runs by default in Expo Go and
  on web. Drives a realistic state machine (scanning → ramp → commit →
  cooldown) so the UI is fully interactive without a dev build. **No
  predictions emit while the camera is off**, so you'll never see ghost
  words spamming when your hand is down.
- **Real mode** (`tfliteRunner.ts → classifyWindow`): wired to
  `react-native-fast-tflite`. Runtime feature-detected — if the native
  module isn't loaded (Expo Go, web, simulator without prebuild), the app
  cleanly falls back to the mock pipeline.

### Wiring real hand landmarks

Hand landmarks need a frame processor on top of `react-native-vision-camera`.
Recommended approach when you build for Android:

1. Drop the official MediaPipe Hand Landmarker `.task` file into
   `assets/models/hand_landmarker.task`. Download from:
   https://developers.google.com/mediapipe/solutions/vision/hand_landmarker#models
2. Load it via `react-native-fast-tflite` the same way the words model is
   loaded.
3. In a `useFrameProcessor` worklet, run hand detection per frame, feed
   landmarks into `extractFrame()` → `normalizeFrame()` →
   `RollingBuffer.push()`. When the FSM says `shouldInfer()`, pass the
   window through `classifyWindow()`.

## Auth — why no MongoDB

The user accounts use `AsyncStorage` for the mockup. **MongoDB requires a
backend server**, which conflicts with the on-device-only architecture.
For production, swap `src/store/authStore.ts` with **Firebase Auth** or
**Supabase** — both give you real auth without standing up a server.
